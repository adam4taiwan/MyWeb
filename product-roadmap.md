# product-roadmap.md

此檔案定義 MyWeb 命理諮詢平台的分階段發佈計劃和功能優先級。

---

## 📅 發佈策略

採用 **MVP (最小可行產品) → 擴展 → 優化** 的漸進式發佈模式。

每個階段約 2-3 個月，完整上線需 12-18 個月。

---

## 🚀 Phase 1: MVP 基礎平台 (第 1-2 個月)

### 目標
建立最核心的功能，實現基本的諮詢業務流程。

### 功能清單

#### 1.1 用戶系統
- [x] 用戶註冊 (郵箱/手機)
- [x] 用戶登入/登出
- [x] 忘記密碼重設
- [x] 基本個人資料編輯 (名字、頭像)
- [x] 用戶認證 (JWT)

#### 1.2 內容展示
- [x] 首頁展示
- [x] 簡介頁面和諮詢師介紹
- [x] 服務說明頁面
- [x] FAQ 頁面
- [x] 聯繫我們頁面

#### 1.3 諮詢管理
- [x] 文字諮詢預約
- [x] 簡單的日歷排程系統
- [x] 諮詢記錄查看
- [x] 基本的評分功能

#### 1.4 點數系統 (簡化版)
- [x] 點數購買頁面
- [x] 單一支付方式 (信用卡 - Stripe)
- [x] 點數顯示和交易歷史
- [x] 簡單的點數扣費邏輯

#### 1.5 基礎功能
- [x] 簡體的通知 (頁面內提示)
- [x] 用戶儀表板
- [x] 基本的 SEO

### 技術棧
- Frontend: Next.js + React 19 + Tailwind CSS (已有)
- Backend: Node.js + Express (新建)
- Database: PostgreSQL + Redis
- Payment: Stripe (信用卡)
- Hosting: Fly.io

### 里程碑
- **Week 1-2**: 後端 API 框架搭建
- **Week 3-4**: 用戶認證和支付集成
- **Week 5-6**: 諮詢管理系統
- **Week 7-8**: 前端集成和測試
- **Week 9-10**: 部署和優化
- **上線**: 第 10 週

### 成功指標
- 平台穩定運作 (99% 可用性)
- 第一個付費用戶
- 用戶反饋收集完成

## 🎯 Concurrent 開發策略

由於您是一人團隊，建議的開發策略：

### Option A: 循序漸進 (推薦)
```
第1-2月:   Phase 1 完整開發和測試
第3-5月:   Phase 2 開發 + Phase 1 優化
第6-8月:   Phase 3 開發 + 前期功能完善
第9-12月:  Phase 4 開發 + 內容充實
第13-18月: Phase 5 開發 + 團隊擴展
```

### Option B: 功能開闢並行
```
同時進行多個小功能，但優先級清晰
優點：快速迭代，快速獲得用戶反饋
缺點：需要良好的代碼組織和測試
```

### 個人開發者的最佳實踐


## ⚠️ 依賴關係和風險

```
Phase 1 (獨立)
    ↓
Phase 2 (依賴 Phase 1)
    ├─ 支付集成
    ├─ 內容系統
    └─ 會員系統
    ↓
Phase 3 (依賴 Phase 1 + 2)
    ├─ AI 功能
    ├─ 課程系統
    └─ 社區功能
    ↓
Phase 4 (依賴 Phase 1-3)
    └─ 擴展和優化

主要風險:
- 支付集成出 issue → 延遲 1-2 週
- 用戶增長慢 → 調整營銷策略
- 技術債積累 → 阻礙後續開發
```

## 🔧 使用外包加快開發

建議在以下地方使用外包服務：

1. **設計** (10-20% 成本)
   - UI/UX 設計師設計主要頁面

2. **支付集成** (10% 成本)
   - 專家協助多渠道支付集成

3. **內容** (5-10% 成本)
   - 初期文章和課程錄製

4. **部署和基礎設施** (5% 成本)
   - DevOps 協助設定監控和自動化
